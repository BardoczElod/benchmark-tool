# React + TypeScript + Vite

This template provides a minimal setup to get React working in Vite with HMR and some ESLint rules.

Currently, two official plugins are available:

- [@vitejs/plugin-react](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react) uses [Babel](https://babeljs.io/) (or [oxc](https://oxc.rs) when used in [rolldown-vite](https://vite.dev/guide/rolldown)) for Fast Refresh
- [@vitejs/plugin-react-swc](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react-swc) uses [SWC](https://swc.rs/) for Fast Refresh

## React Compiler

The React Compiler is not enabled on this template because of its impact on dev & build performances. To add it, see [this documentation](https://react.dev/learn/react-compiler/installation).

If you are developing a production application, we recommend updating the configuration to enable type-aware lint rules:

```js

  globalIgnores(['dist']),

      // Optionally, add this for stylistic rules
      tseslint.configs.stylisticTypeChecked,

      // Other configs...
    ],

      parserOptions: {
        project: ['./tsconfig.node.json', './tsconfig.app.json'],
        tsconfigRootDir: import.meta.dirname,
      },
      // other options...
    },
  },
])
```

## Running in Docker

A multi-stage Dockerfile is provided which builds the app with Node 20 and serves the static `dist/` via Nginx.

### Local Development with Docker

Build the image:

```bash
docker build -t gmb-ecc-report:latest .
```

Run with Docker:

```bash
docker run --rm -p 5173:80 gmb-ecc-report:latest
```

Or use docker-compose (recommended):

```bash
docker-compose up --build
```

Then open: http://localhost:5173/

### Production Server Deployment

For production deployment on a server, see **[DEPLOYMENT.md](./DEPLOYMENT.md)** which includes:

- Environment configuration
- Reverse proxy setup (Nginx/Traefik)
- SSL/HTTPS configuration
- Health checks and monitoring
- Security best practices
- CI/CD automation examples

Quick production deployment:

```bash
# On your server
cp .env.example .env
# Edit .env to set HOST_PORT (e.g., 80 for production)
docker-compose up -d --build
```

### Docker Features

- **Multi-stage build** - Optimized image size (~25MB final image)
- **Health checks** - Built-in health monitoring at `/health`
- **Security** - Runs as non-root user with security headers
- **Logging** - Configured log rotation (10MB max, 3 files)
- **Caching** - Aggressive static asset caching
- **Compression** - Gzip enabled for optimal performance

Notes:
- The Docker image builds using Node 20 and serves content via Nginx
- Ensure Docker Desktop or Docker Engine is installed on your machine
- For production, always use HTTPS with a reverse proxy
